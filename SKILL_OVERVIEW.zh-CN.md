# Print Studio OpenDesign · Skill 详细说明

> 这份文档用于先把这个 Skill 自己讲清楚：它解决什么问题、怎么工作、能交付什么、哪些地方可以自定义、哪些地方还只是 V1 边界。

---

## 1. 一句话介绍

Print Studio OpenDesign 是一个给印刷品系列打样使用的 Codex / Claude Code Skill。它可以把一组插画或图片，按照预设的产品类型、尺寸、纸张材质、系列结构和印刷规则，自动生成可确认的版式预览、印刷文件、白底图、氛围图、系列总览图、单张 4K 下载图、材质选择页、质检报告和交付压缩包。

更短一点说：

> 它是一个“印刷品 AI 打样工作台”：先选产品规格，再让 AI 在规则内排版，最后输出展示图和印刷交付包。

---

## 2. 它不是单纯的日历工具

最开始的场景是插画日历，但现在定位已经升级成“印刷品系列模板 Skill”。

它可以覆盖：

- 贺卡
- 明信片
- 桌历 / 月历卡
- 挂历
- 季度卡
- 节日套装
- 卡牌套装
- 书签
- 感谢卡
- 邀请卡
- 包装插卡
- 礼品吊牌
- 贴纸卡
- 自定义 N 张系列

日历只是其中一个模板，真正核心是：

- 选择产品类型
- 选择尺寸
- 选择纸张 / 材质
- 选择系列张数
- 选择工艺和结构
- 生成可确认的视觉稿
- 生成可检查的印刷文件

---

## 3. 这个 Skill 解决的实际问题

印刷品定制打样常见的问题是：

- 图像有了，但不知道怎么快速排成一套。
- 每个项目尺寸、纸张、张数都不同，纯手工排版很耗时间。
- 屏幕预览好看，印刷后可能偏灰、偏暗、字太浅。
- 展示图和印刷源文件经常混在一起，容易误用。
- 需要给确认方看效果，但又不能一上来就做完整设计成本。
- 材质很多，白卡、蓝芯、黑芯、珠光、布纹、PVC 等不容易快速比较。

这个 Skill 的思路是：

- 用配置文件锁定产品规格。
- 用固定规则保证出血、安全边距、DPI、日期、文件结构。
- 让 AI 只在“审美排版”和“展示图生成”里发挥。
- 输出多个层级的文件，分别用于确认、展示、印刷和质检。

## 4. 整体工作流

```text
导入插画 / 图片
        │
        ▼
选择产品类型、尺寸、纸张、系列结构
        │
        ▼
先生成材质选择页和快速预览
        │
        ▼
确认规格后运行完整打样
        │
        ├─ screen/            屏幕预览
        ├─ print/             印刷文件
        ├─ commerce/          白底图 / 氛围图 / 系列总览
        ├─ download_4k/       单张 4K 下载
        ├─ provider_previews/ 三模型首轮预览
        ├─ qc_report.json     自动质检
        └─ *_交付包.zip       打包文件
```

## 5. 两种交付形态

仓库里有两套结构：

### 5.1 独立 Skill

路径：

```text
print-studio-opendesign/
```

适合简单安装，作为一个完整 Skill 给 Codex 调用。

### 5.2 插件套件

路径：

```text
plugins/print-studio-opendesign/
```

这是更接近 OpenDesign 的交付方式：一个插件里拆成多个小 Skill，每个 Skill 对应一个动作。

| 小 Skill | 作用 |
| --- | --- |
| `print-studio-order-intake` | 把需求整理成配置 |
| `print-studio-material-selector` | 生成材质选择页 |
| `print-studio-layout-proof` | 做卡片、书签、吊牌、明信片排版 |
| `print-studio-calendar-series` | 做月历、季度、节日系列 |
| `print-studio-commerce-mockup` | 做白底图、氛围图、4K 下载、模型预览 |
| `print-studio-prepress-qc` | 做印前检查和交付包 |
| `print-studio-runtime` | 共用渲染引擎 |

实际推荐使用插件套件，因为它更像一个完整产品，而不是一个单文件脚本。

## 6. 配置文件怎么理解

核心配置在：

```text
config.example.json
examples/*.json
```

可以把配置理解成一张“电子接单表”。

### 6.1 `job`

记录项目基础信息：

- 项目方名称
- 项目名称
- 主题名称
- 备注

这些信息主要用于输出目录、文件命名和留痕。

### 6.2 `preset`

决定产品规格：

- 产品类型：贺卡、明信片、挂历、书签等
- 尺寸：5×7、7×5、2×6、8×12 等
- 材质：白卡、牛皮、珠光、蓝芯、黑芯、PVC 等
- 产品形态：单卡、挂历、吊牌、书签等
- 视觉风格：复古植物志、高级留白、温柔插画等

### 6.3 `series`

决定一套里面有多少张：

- 单张
- 4 张四季度
- 6 张节日套装
- 12 张月历
- 自定义 N 张

### 6.4 `content`

决定文字内容：

- 主标题
- 副标题
- 每张卡片标题
- 月份文字
- 年份
- 周起始
- 印章 initials
- 角落短句

### 6.5 `production`

决定生产结构：

- 是否双面
- 是否圆角
- 圆角半径
- 是否吊牌打孔
- 挂历是否顶部线圈 / 挂孔 / 夹条
- 绑定预留区
- 出血和安全边距

### 6.6 `outputs`

决定输出什么：

- `screen`：屏幕预览
- `print`：印刷文件
- `commerce`：展示图
- `download_4k`：单张 4K 保存图
- `whitebg`：白底图
- `ambiance`：氛围图
- `grid`：系列总览

### 6.7 `image_gen.provider_previews`

决定是否首轮比较图像模型：

- 即梦
- Gemini
- GPT-Image2

默认不开启，避免一运行就消耗模型额度。需要比较时，把 `enabled` 改成 `true`。

## 7. 输出文件怎么看

完整运行后，会在 `output/<项目名>/` 下生成结果。

| 输出 | 用途 | 说明 |
| --- | --- | --- |
| `preview/preview.html` | 快速预览 | 看尺寸、材质、版式、印刷 guide |
| `preview/materials.html` | 材质选择 | 一次看多种纸张 / PVC 模拟效果 |
| `screen/` | 屏幕确认 | 颜色更柔和，适合线上看 |
| `print/` | 印刷生产源文件 | 300dpi，含出血、安全边距、裁切 guide |
| `commerce/` | 展示图 | 白底图、氛围图、系列总览图 |
| `download_4k/` | 单张保存 | 每张图长边 4096px，可逐张下载保存 |
| `provider_previews/` | 模型对比 | 即梦 / Gemini / GPT-Image2 首轮预览 |
| `qc_report.json` | 自动质检 | 检查尺寸、边距、日期、输出完整度 |
| `*_交付包.zip` | 打包交付 | 把核心输出文件压成一个包 |

重点要记住：

- 印刷用 `print/`。
- 展示用 `commerce/`。
- 一张张保存用 `download_4k/`。
- 模型比较看 `provider_previews/`。
- 不要把展示图当印刷源文件。

---

## 8. 白底图、氛围图、系列总览图是什么

### 8.1 白底图

白底图用于干净展示产品本身，适合：

- 商品页
- 报价确认
- 款式归档
- 系列清单

特点：

- 背景干净
- 产品边缘清楚
- 颜色可以比印刷源文件更亮一点
- 但不能改掉卡片内容和形状

### 8.2 氛围图

氛围图用于把产品放进场景里，适合：

- 提案展示
- 社媒展示
- 视觉确认
- 风格判断

特点：

- 可以有桌面、花、布料、光影等道具
- 但不能改变卡片版式、日期、文字和插画主体

### 8.3 系列总览图

系列总览图用于把多张排在一起看，例如：

- 12 个月花卉日历一起展示
- 四季度卡一起展示
- 6 张节日卡一起展示
- 一套书签一起展示

它的作用是判断整套是否统一：

- 色调是否一致
- 标题层级是否一致
- 插画大小是否一致
- 留白是否均衡
- 系列感是否成立

## 9. 4K 单张下载规则

现在 Skill 会生成：

```text
download_4k/index.html
download_4k/B_01_4k.jpg
download_4k/C_series_4k.jpg
...
```

规则：

- 每张图都保存为长边 4096px。
- 页面里每张图都可以单独点击下载。
- 适合逐张保存、发图确认、归档展示。
- 4K 下载图是展示/确认版本，不替代 `print/` 印刷文件。

这点很重要：别人要一张张存图时，不需要去 ZIP 里找，也不需要自己截图。

---

## 10. 三模型首轮预览规则

首轮可以提供三种图像模型对比：

- 即梦
- Gemini
- GPT-Image2

输出位置：

```text
provider_previews/index.html
```

建议用法：

1. 先只选 1–2 张图。
2. 白底图和氛围图都生成。
3. 对比三个模型哪个更稳定。
4. 选定模型后，再批量生成完整展示图。

为什么不默认全开？

- 三模型都跑会消耗额度。
- 即梦接入方式可能跟账号/API 网关有关，需要按实际环境补适配。
- 首轮一般只需要比较风格，不需要全量跑完。

如果缺少 API key 或某个模型没接好，系统会把错误写进 `provider_previews/index.html`，流程不会直接崩掉。

## 11. 材质预览规则

当前支持多种纸张 / 材质模拟，包括：

- 300g 超白白卡纸
- 牛皮卡纸
- 冰白珠光纸
- 莱妮纹
- 蛋壳纹
- 皱纹纸
- 沙龙纹
- PVC 光面 / 磨砂
- 白芯纸
- 蓝芯纸
- 黑芯纸

预览会尽量模拟：

- 纸张底色
- 纹理强弱
- 珠光反光
- PVC 高光
- 蓝芯 / 黑芯 / 白芯侧边
- 小字风险提示
- 浅色图风险提示

但必须注意：

> 材质预览是屏幕模拟，不是最终实物保证。最终仍要以纸样和打样为准。

## 12. 印刷颜色规则

这个 Skill 内置的基本判断是：

- 屏幕图可以轻、灰、淡。
- 印刷图不能太“仙”。
- 印刷后通常会更暗、更灰。
- 所以印刷版要比屏幕版略提高饱和度和对比。
- 但不能变成廉价高饱和，要做“高级饱和”。

具体规则：

- 主体插画可以适度增强。
- 背景米白要偏暖白、奶油白、浅象牙白，避免发脏。
- 日期数字不能太浅。
- 主视觉和背景要有足够明度差。
- 粉色、黄色、浅绿色要重点检查。
- 黑色文字优先深棕 / 深灰棕，不建议纯黑。
- 细线、印章、手写小字不能太细太浅。
- 重要内容不能贴边，要保留裁切安全区。

## 13. AI 在哪里参与，哪里不能乱动

AI 可以参与：

- 判断主体图放大还是缩小
- 判断标题层级
- 判断留白比例
- 生成白底图 / 氛围图
- 生成首轮模型预览
- 给配置建议
- 给材质选择建议

AI 不能随便改：

- 成品尺寸
- 出血
- 安全边距
- 日期计算
- DPI
- 印刷导出结构
- QC 规则
- 印刷源文件内容

一句话：

> AI 负责审美和提案效率，系统规则负责印刷安全。

## 14. 实际怎么跑

进入 runtime：

```bash
cd plugins/print-studio-opendesign/skills/print-studio-runtime
```

先跑材质预览：

```bash
python scripts/preview_config.py examples/greeting-card-5x7.json --all-materials
```

再跑完整交付：

```bash
python scripts/run.py examples/greeting-card-5x7.json
```

如果要做自己的项目：

```bash
cp examples/greeting-card-5x7.json jobs/my-card.json
python scripts/run.py jobs/my-card.json
```

主要改：

- `job.job_name`
- `theme`
- `illustrations_dir`
- `preset.product_type`
- `preset.size`
- `preset.material`
- `series.count`
- `content`
- `production`
- `outputs`

- 产品类型
- 尺寸
- 材质
- 系列张数
- 标题
- 副标题
- 年份
- 周起始

- 出血 mm
- 安全边距 mm
- 顶部装订预留区
- 色彩补偿
- 材质补偿
- 自定义尺寸
- 自定义风格

不建议普通使用者乱改：

- 渲染代码
- DPI
- 日期逻辑
- QC 阈值
- 材质 key
- 输出目录结构

## 17. 当前边界和不能承诺的事

V1 不承诺：

- 真实 CMYK / ICC 色彩管理
- 不同印刷机的精确色彩补偿
- 生产级烫金 / UV / 压纹模拟
- ERP / 报价系统 / 排产系统集成
- 多用户 Web 后台
- 所有产品类型都有完全独立的高精模板
- 即梦 API 已完全接入所有账号环境

更准确的说法是：

> V1 适合快速打样、确认、展示和初步印刷文件生成；批量生产前仍建议实物打样和印前人工复核。

## 18. 推荐怎么向别人解释

可以这样介绍：

> 这是一个可安装到 Codex / Claude Code 的印刷品 AI 打样 Skill。它支持选择产品类型、尺寸、纸张材质、系列张数和基础工艺，然后把一组插画自动排成贺卡、明信片、日历、挂历、书签、吊牌等系列设计稿。输出会分成屏幕预览、300dpi 印刷文件、白底图、氛围图、系列总览图、单张 4K 下载图、材质选择页、三模型首轮预览、QC 报告和交付 ZIP。AI 负责在规则内做美观排版和展示图，出血、安全边距、DPI、日期和质检由确定性规则兜底。

如果要更短：

> 它是给印刷品定制打样用的 AI 工作台：选规格、导素材、自动排版、出预览、出印刷文件、出展示图和 4K 下载包。

## 19. 下一步最值得补什么

建议下一步优先补：

1. 更清晰的 PDF 导出规则。
2. 即梦真实 API 适配。
3. 常用纸张的实测色彩补偿参数。
4. 双面卡片的生产级拼版规则。
5. 烫金 / UV / 压纹的可视化但不冒充生产级模拟。
6. 更漂亮的 Web 预览入口，把配置、材质、输出集中到一个页面。
